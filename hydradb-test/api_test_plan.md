# HydraDB Raw API Test Plan

**Purpose:** verify HydraDB's documented claims at the raw HTTP layer (no SDK), complementing the SDK-based testing done in `stock-market-decoder`. Every test maps to a specific claim in `hydradb_claims_extraction.md` (crawled from hydradb.com / docs.hydradb.com on 2026-07-12).

**Setup:** tenant/database `sia-test-2` (fresh, created in T05; `sia-test` was used for Phase 0 reads only), base URL `https://api.hydradb.com`, auth `Authorization: Bearer $HYDRA_DB_API_KEY` + `API-Version: 2` header. Credentials in `.env` (this folder). Tests live as `%%bash` cells in `test.ipynb`, run one at a time; verdicts get recorded next to each test.

**Datasets (synthetic, this folder):**
- `rfc_project_atlas_migration.pdf` — RFC-019, approved 2026-03-04: migrate Project Atlas similarity search to a vector DB, decommission Postgres.
- `meeting_notes_atlas_rollback.pdf` — 2026-04-22 review: security incident, decision **partially reversed** (keep Postgres for permission re-filtering; p95 45ms → 120ms).

Shared entities across both docs (Wallace Okafor, Marcus Webb, Priya Chandrasekaran, Alex Kim, Project Atlas, RFC-019, Postgres) + a decision that gets superseded → ideal for entity-resolution and temporal/knowledge-update claims.
